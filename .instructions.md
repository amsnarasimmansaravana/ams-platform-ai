# AMS Platform AI - Workspace Instructions

## Project Context
- **Project Name**: AMS Platform AI
- **Focus**: Architecture and AI platform documentation
- **Key Artifacts**: PlantUML diagrams for architecture, business models, and deployment patterns

## How to Work With Me

### For Documentation
- Prioritize clarity and completeness in technical documentation
- Cross-reference architecture decisions with the ADRs in `docs/templates/`
- Maintain consistency with the existing glossary in `docs/v1.0/06-GLOSSARY.md`
- Use the requirement templates from `docs/templates/REQUIREMENT-TEMPLATE.md`

### For Diagrams (PlantUML)
- When modifying diagrams, preserve existing diagram relationships
- Use consistent styling across all diagram files in `docs/diagrams/`
- Reference the architecture documentation when making diagram changes
- Preview diagrams using the PlantUML Viewer extension (Cmd/Ctrl + Shift + P → "PlantUML: Preview")

### For Code & Configuration
- Store any generated scripts in appropriate directories
- Follow existing naming conventions in the project
- Create small, focused changes rather than attempting large rewrites
- Test any generated code before committing

### Communication Style
- Be concise and direct
- Show file paths as links with line numbers when relevant
- Use markdown formatting for clarity
- Ask clarifying questions when requirements are ambiguous

## Project Structure
```
packages/ams-common/    # Shared Python library (config, logging, middleware)
services/gateway/       # API Gateway (reverse proxy)
services/*-service/     # Domain microservices (agent, orchestration, tool, execution)
docker-compose.yml      # Full stack (gateway + services + Postgres + Redis)
docs/
  ├── README.md
  ├── diagrams/ (PlantUML files for visualization)
  ├── templates/ (ADR and requirement templates)
  └── v1.0/ (Documentation for version 1.0)
frontend/          # Placeholder for React/Tauri (see frontend/README.md)
docker-compose.yml # Optional Postgres + Redis + API
```
